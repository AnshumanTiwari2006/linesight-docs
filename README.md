Welcome to the documentation repository for **LineSight**! 

This repository hosts the static front-end code that powers our official landing page and API documentation. If you are looking for the actual Python library source code, please visit our main repository here: [**LineSight Python Library**](https://github.com/AnshumanTiwari2006/linesight).

---

## 🌟 About the Website

The LineSight documentation site is built to reflect the core philosophy of our library: **complete transparency**. 

Instead of relying on heavy frameworks, this site is built from scratch using pure HTML5, CSS3, and Vanilla JavaScript. It is designed to be 100% static, incredibly fast, and seamlessly integrated with our pedagogical visualizations.

### What you will find here:
- **Interactive Documentation:** A dynamic documentation engine that pairs our API methods directly with mathematical concept diagrams.
- **Visual Galleries:** Complete galleries of the 3D loss surfaces, residual plots, and training animations generated by the library.
- **Tutorial Downloads:** Direct access to our `LineSight_Interactive_Tutorial.ipynb` for hands-on learning.

## 🛠️ Running the Site Locally

If you would like to contribute to the documentation UI or test changes locally, you can do so in seconds. Because there are no backend dependencies or build steps, you don't even need Node.js!

1. Clone this repository:
   ```bash
   git clone https://github.com/AnshumanTiwari2006/linesight-docs.git
   ```
2. Open the folder in your favorite code editor.
3. Open `index.html` directly in your web browser. 
*(For the best development experience, we recommend using the **Live Server** extension in VS Code).*

---

## 📝 Editing the Documentation Content

If you are a contributor looking to update the documentation text, you **do not** need to edit the HTML files. 

All documentation content (descriptions, code examples, parameters) lives inside a centralized database file:
📍 `docs/v0.1.0/api-data.js`

Our custom JavaScript engine (`docs.js`) automatically reads this file and generates the UI structure on the fly!
